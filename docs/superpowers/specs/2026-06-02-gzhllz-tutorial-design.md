# 微信公众号流量主实战教程网站设计规约 (WeChat Traffic Master Tutorial Design Spec)

*   **创建日期**：2026-06-02
*   **状态**：草案 (Draft)

---

## 1. 项目概述 & 背景

本项目是一个针对零基础新手的**微信公众号流量主实战教程**网页网站。旨在将公众号流量主的实操干货（起号、AI写作、相似度检测、避坑、防限流）通过结构化、递进式的章节呈现给用户。

在视觉与交互上，本项目将深度参考 **Apple 风格的视觉设计规范**，并结合 **OpenCode 的系统化课程架构**，构建一个兼具营销展示力与深度学习体验的精品教程网站。

---

## 2. UI 与交互设计规范 (基于 Apple 风格)

设计系统基于 `DESIGN.md` 中定义的 Apple 规范，通过极致的版式对比、无衬线字体及标志性的蓝色交互点缀，营造 Premium（高端、专业）的质感。

### 2.1 颜色系统 (Color Palette)
*   **深色画布**：`#000000` (用于首页 Hero 区域、特色展示等高对比、高质感模块)
*   **浅色画布**：`#f5f5f7` (用于文档页侧边栏背景、首页功能区切换等)
*   **明亮画布**：`#ffffff` (主要用于文章正文阅读区、产品卡片列表，确保极佳的文字可读性)
*   **主要文本**：`#1d1d1f` (高对比度深灰，避免纯黑字体的刺眼感)
*   **次要文本**：`#6e6e73` (用于辅助说明、发布日期、作者等元信息)
*   **行动指向蓝**：`#0071e3` (核心 CTA 按钮背景、聚焦状态、主交互文字)
*   **正文链接蓝**：`#0066cc` (文章正文内嵌超链接，提供优秀的对比度与可读性)
*   **细边框与线条**：`#d2d2d7` (淡灰色，用于卡片边框、正文分割线等)

### 2.2 字体排版规则 (Typography)
*   **全局英文字体**：`Inter` (若系统支持则优先调用 `SF Pro Text` / `SF Pro Display`)
*   **大标题 (Display)**：`Inter Tight`，加粗 (Semibold, 600)，字距紧凑 (-0.02em)，行高 1.05~1.1。
*   **正文文本 (Body)**：`Inter`，字重 400，字号 17px，行高 1.5，字距微调，确保长文本阅读的呼吸感与舒适度。

### 2.3 圆角阶梯体系 (Border Radius Scale)
*   **8px**：按钮、输入框、小标签、控制开关。
*   **18px**：内容卡片、弹窗容器、配置面板、侧边栏卡片。
*   **36px / Capsule (胶囊型)**：首页主视觉卡片、浮动控制栏、大尺寸 CTA 动作按钮。

## 3. 页面结构与核心功能

### 3.1 首页 (Home Page)
*   **磨砂玻璃导航栏 (Navigation Bar)**：顶部固定，背景为毛玻璃半透明效果 (`backdrop-filter: blur(20px)`)，包含 LOGO、六个阶段学习快捷入口以及“关于作者/友情链接”。
*   **苹果黑 Hero 视界 (Hero Block)**：深邃的 `#000000` 背景，搭配巨幅紧凑的银白色大字标题（“零基础，靠阅读变现——2026公众号流量主全体系教程”），配以双极简胶囊按钮（“立即学习”、“浏览大纲”）。
*   **六大阶段成长卡片网格**：展示 1-6 阶段的递进卡片。每个卡片采用 18px 圆角，轻微的悬浮缩放动画效果 (Hover micro-animations)。
*   **场景实战路线展示 (Showcase)**：直观对比“退休鸡汤文”、“美食指南”、“老歌漫画”三大赛道的实操卡片，点击可直接跳入第四阶段相应的实战小节。

### 3.2 文档学习页 (Doc Page)
*   **左侧导航侧边栏 (Sidebar)**：
    *   背景为浅灰色 (`#f5f5f7`)，可折叠/收缩以最大化正文阅读区。
    *   按照 6 大阶段呈现树状大纲。支持点击子标题折叠/展开非当前激活阶段。
*   **中间正文区 (Content Canvas)**：
    *   纯白背景 (`#ffffff`)，两侧留白符合 Apple Showcase 比例。
    *   Markdown 内容转译后，保持严谨的排版：引言高亮、代码块（带有一键复制按钮）、数据表格符合苹果简洁线条风。
*   **右侧文章大纲 (Table of Contents)**：
    *   浮动展示当前文章的 H2 / H3 标题锚点链接。
    *   支持滚动监听 (Scrollspy)，当前阅读章节标题高亮展示。

## 4. 技术与构建架构 (与 OpenCode 保持对齐)

为确保高效的开发调试以及极佳的 SEO 性能，网站使用以下构建方案：

```
                        [ docs/ 目录下的 .md 源文件 ]
                                     │
                                     ▼ (node scripts/convert.js)
                             [ src/data/pages.json ]
                                     │
                 ┌───────────────────┴───────────────────┐
                 ▼ (开发模式)                            ▼ (生产构建)
        [ webpack-dev-server ]                  [ webpack 生产编译 ]
                 │                                       │
                 ▼                                       ▼
          [ 动态 SPA 渲染 ]                         [ dist/ 静态 JS/CSS ]
                                                         │
                                                         ▼ (node scripts/build-static.js)
                                                    [ 生成每个页面的独立静态 HTML ]
```

### 4.1 核心脚本职责
1.  `convert.js`：递归读取 `docs/` 下的六个阶段文件夹，提取 markdown 文件头部 metadata (YAML format) 和正文内容，并将它们整合转换为统一的 JSON 配置文件 `src/data/pages.json`。
2.  `build-static.js`：在 production 构建后，提取打包生成的 React App 代码，在服务端使用 `react-dom/server` (SSR 思想) 渲染出每一个文档路径对应的完整、包含实际文字的独立静态 `.html` 文件，从而达成“极致首屏速度”与“搜一搜抓取友好 (SEO)”的商业目的。

## 5. 验证与测试计划

### 5.1 自动化测试 (命令行与浏览器)
*   运行 `pnpm run build` 验证打包是否通过，且未出现 TypeScript 及打包编译报错。
*   使用静态服务部署打包后的 `dist` 目录，在桌面端和移动端浏览器中检查：
    1.  首屏加载时间与 HTML 结构中是否已嵌入实际的 Markdown 翻译文字（保证 SEO）。
    2.  侧边栏展开、折叠及大纲折叠的响应。
    3.  右侧 TOC 锚点跳转与滚动监听激活效果。

### 5.2 手动设计审查 (Design Check)
*   **色彩核对**：检查是否有任何地方混入了杂乱的次要彩色，确保除了 Action Blue 之外的 UI 控件保持极简黑白灰。
*   **移动端适配**：拖动浏览器宽度至 375px，检查左侧导航栏是否能优雅地隐藏进汉堡菜单/侧边抽屉，正文间距是否自适应缩紧。
*   **AI味消除与体验**：随机阅读 3 篇文档，验证 markdown 渲染排版的美观度（呼吸感）。
