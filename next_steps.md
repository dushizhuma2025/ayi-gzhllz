# 🤖 大模型接力指南：如何补全剩余的公众号流量主教程页面

你好，接班的 AI 伙伴！这是一个成体系的“微信公众号流量主实战教程”网页网站项目。
为了帮助用户高效地补全剩下的教程页面，你需要按照以下技术架构与文件对照表，依次将 `原始资料/` 中的草稿文件转化并导入为 `docs/` 目录下的系统教程 Markdown 页面，并激活侧边栏超链接。

---

## 🛠️ 项目技术栈与运转逻辑

本站基于 **React 18 + Webpack 5 + TypeScript + React Router 6 + pnpm** 架构搭建。

1. **教程渲染核心机制**：
   - 所有的教程正文（Markdown 文件）都存放在根目录下的 `docs/` 目录中。
   - 编译时，脚本 `node scripts/convert.js` 会扫描 `docs/` 目录下的所有子文件夹，解析 Markdown 的 YAML Frontmatter（标题、描述等），并使用 `marked` 解析库将正文转换为 HTML 结构，最后汇聚生成统一的数据源 **`src/data/pages.json`**。
   - 前端采用高内聚动态组件 `src/components/ArticleView.tsx` 作为通用渲染器，匹配当前路由在 `pages.json` 中查找对应的教程正文渲染。
2. **侧边栏解锁机制**：
   - 侧边栏的配置文件位于 **`src/data/sidebar.ts`**。
   - 对于尚未开放的页面，侧边栏菜单项仅配置了 `text` 字段（此时前端会被高亮置灰，不可点击）。
   - **一旦你补全了页面，请为 `src/data/sidebar.ts` 对应的项加上 `link: '/路径'` 属性，即可立即解锁侧边栏点击与路由跳转。**
3. **已运行服务**：
   - 本地开发服务器已拉起在：`http://localhost:5173/`。

## 📝 编写 Markdown 文件格式规范

每次写入 Markdown 时，**文件顶部必须包含 YAML Frontmatter 标头**。示例模板如下：

```markdown
---
title: "1.2 入池概念深度拆解"
description: "介绍公众号流量主推流的入池原理与数据判定"
tags: "微信公众号, 流量主, 入池"
created: "2026-06-02"
---

# 1.2 入池概念深度拆解：什么是“入池”？

这里是正文部分，你需要：
- 转换并优化草稿，剔除冗余口语。
- 使用标准的 Markdown 标题（用 ## 和 ###）来构造段落，这会自动被前端渲染为好看的排版。
- 在逻辑重点处可以使用 `> [!NOTE]` 或标准的 blockquote 进行高亮提醒。
- 将草稿中的大白话公式用代码块（如 \`\`\`）或者表格输出以提高精致感。
```

## 🚀 补全操作三步走指南

当你开始着手某一个教程页面（例如：`1.2 入池概念深度拆解`）时，只需运行以下工作流：

1. **第一步：编写 Markdown 文件**：
   - 按照上面的映射表，读取 `原始资料/公众号流量主教程第2课：了解公众号入池概念.md`。
   - 在 `docs/1-start/` 下新建 `03-concept.md` 文件并写入转化后的精美教程排版，确保包含 YAML Frontmatter。

2. **第二步：激活侧边栏超链接**：
   - 打开 `src/data/sidebar.ts`，定位到对应的项：
     ```typescript
     { text: '1.2 入池概念深度拆解' }
     ```
   - 修改它，补齐 `link` 字段：
     ```typescript
     { text: '1.2 入池概念深度拆解', link: '/1-start/03-concept' }
     ```

3. **第三步：运行脚本生成数据**：
   - 在终端根目录下运行如下命令：
     ```bash
     node scripts/convert.js
     ```
   - 这会扫描你的 Markdown 文件并生成最新的 `src/data/pages.json`。
   - 开发服务器（已在后台运行）会立即监听到 `pages.json` 的更新，自动实现热重载刷新浏览器，侧边栏按钮直接解锁，点击即可查看新解锁的文章！

4. **第四步：打包验证**：
   - 当完成阶段性导入后，运行 `pnpm run build` 命令验证整个工程的类型检查与生产静态生成有没有任何错误。
   - 验证通过后即可使用 Git 归档。
